# Analysis of Xenium In Situ Data Using Squidpy and SpatialData


---

## Introduction

This analysis presents a complete workflow for processing and analyzing data generated by the **10x Genomics Xenium In Situ** platform using a combination of Squidpy, Scanpy, and the SpatialData ecosystem. Xenium is a high-resolution, in situ transcriptomics technology that detects and spatially localizes individual RNA transcripts within intact tissue sections at subcellular resolution, enabling single-cell gene expression profiling while preserving precise spatial context.

The dataset used is a publicly available Xenium profiling of **FFPE (formalin-fixed paraffin-embedded) human lung cancer** tissue, which provides an ideal context for spatial analysis given the complex and heterogeneous cellular architecture characteristic of tumors. The dataset is exceptionally large — **161,000 single cells** profiled across **480 genes** — with over 40 million individual transcript detections recorded across the tissue.

A key distinguishing feature of this tutorial relative to other Squidpy workflows is the use of the **SpatialData** framework as the primary data container. SpatialData is a modern, unified data standard for spatial omics that stores images, cell segmentation labels, transcript point clouds, cell boundary shapes, and gene expression tables in a single coherent object backed by the Zarr storage format. This enables efficient on-disk access to large datasets without loading everything into memory.

The computational environment uses **SpatialData** and **spatialdata-io** for data ingestion and storage, **Scanpy** for gene expression preprocessing and clustering, **Squidpy** for spatial graph construction and statistical analysis, **Seaborn** and **Matplotlib** for visualization, and **spatialdata-plot** for overlay visualization of gene expression on the morphology image. An optional interactive visualization component using **napari-spatialdata** is also demonstrated.

---

## 1. Data Loading and the SpatialData Structure

### 1.1 Reading Xenium Data

The Xenium output directory — produced by the 10x Genomics Xenium instrument — is read using the `xenium()` reader function from the `spatialdata-io` package. This reader parses all components of the Xenium output (images, transcript coordinates, cell segmentations, and count matrices) and assembles them into a single `SpatialData` object. The object is then written to disk in **Zarr format** — a chunked, compressed, hierarchical storage format optimized for large array data — and all subsequent operations read directly from the Zarr store rather than loading the full dataset into memory.

### 1.2 SpatialData Object Structure

The resulting SpatialData object contains five categories of data, all registered to a shared global coordinate system:

- **Images:** A single multi-scale morphology focus image (`morphology_focus`) stored at five resolution levels — from the full-resolution 20,535 × 51,103 pixel image down to a highly downsampled thumbnail. The multi-scale pyramid enables efficient rendering at different zoom levels without loading the full image.

- **Labels:** Two multi-scale label images — `cell_labels` and `nucleus_labels` — each a pixel-level segmentation mask where every pixel belonging to a given cell or nucleus is assigned that cell's integer identifier. These were generated by the Xenium instrument's built-in cell segmentation pipeline.

- **Points:** A DataFrame of **40,257,199 individual transcript detections** (`transcripts`), each described by its 3D spatial coordinates (x, y, and z depth within the tissue section) along with gene identity and quality metrics. This is the primary raw output of the Xenium platform.

- **Shapes:** Three GeoDataFrames covering all 161,000 detected cells — `cell_boundaries` (precise polygon outlines of each cell), `nucleus_boundaries` (polygon outlines of each nucleus), and `cell_circles` (simplified circular approximations of cell positions used for efficient spatial queries).

- **Tables:** An AnnData object (`table`) containing the **per-cell gene expression count matrix** (161,000 cells × 480 genes) along with per-cell metadata and spatial coordinates.

### 1.3 AnnData Table Content

The AnnData table extracted from the SpatialData object contains rich per-cell metadata for all 161,000 cells, including: unique cell identifiers, raw transcript counts, control probe counts (negative DNA probes used to estimate background noise), control codeword counts (decoding errors), unassigned and deprecated codeword counts, total counts, cell area, nucleus area, z-level of the focal plane, and nucleus count per cell. Cell spatial coordinates (centroid positions) are stored in the `obsm["spatial"]` slot and are automatically populated by the Xenium reader, making the data immediately compatible with Squidpy's spatial analysis functions.

## 2. Quality Control

### 2.1 Control Probe Assessment

Before inspecting per-cell metrics, two dataset-level quality indicators are computed from the control metadata:

- **Negative DNA probe count percentage:** The fraction of total counts attributed to negative control DNA probes — sequences designed to hybridize to nothing in the tissue. A low value indicates minimal non-specific background binding. In this dataset, this value is approximately **0.005%**, indicating very low non-specific signal.

- **Negative decoding count percentage:** The fraction of total counts arising from decoding errors during fluorescence readout. This value is approximately **0.003%** — again extremely low — suggesting high fidelity in the combinatorial barcoding and decoding process used by Xenium.

These two metrics collectively confirm that the dataset has very high signal-to-noise characteristics at the platform level.

### 2.2 Per-Cell Quality Metrics

Standard QC metrics are calculated using Scanpy's `calculate_qc_metrics()` function, which computes additional per-cell statistics including the number of unique genes detected and the percentage of counts in top-expressed genes. Four histograms are plotted side by side to visualize the quality distribution across all 161,000 cells:

- **Total transcripts per cell:** The distribution of total RNA counts per cell, reflecting how much gene expression was captured per segmented cell. This informs the lower threshold for filtering cells with insufficient signal.

- **Unique transcripts per cell (n_genes_by_counts):** The number of distinct genes detected per cell, reflecting the complexity of expression captured. Cells with very few unique genes may be low-quality or represent debris.

- **Area of segmented cells:** The physical size of each segmented cell in the tissue. Very small cells may represent over-segmented fragments, while extremely large cells may be under-segmented doublets or artifact regions.

- **Nucleus ratio (nucleus area / cell area):** The fraction of the total cell area occupied by the nucleus. This metric can help identify cells with unusual morphology — very high ratios might indicate bare nuclei without cytoplasm captured, while very low ratios could reflect cells where the nucleus was not properly segmented.

These four plots collectively guide the choice of filtering thresholds in an empirically informed way.

### 2.3 Filtering

Based on the distributions observed in the QC plots, cells are filtered using a minimum count threshold of 10 transcripts (removing cells with insufficient RNA capture), and genes are filtered using a minimum cell detection threshold of 5 cells (removing genes that are effectively absent from the dataset). These are intentionally conservative thresholds appropriate for a targeted 480-gene panel, where every gene on the panel is expected to be biologically relevant.

---

## 3. Preprocessing, Dimensionality Reduction, and Clustering

Before normalization, the raw integer count matrix is preserved by copying it into a separate layer within the AnnData object (`adata.layers["counts"]`). This is important because downstream normalization and log transformation modify the primary expression matrix, and retaining the raw counts ensures they remain accessible for any analyses that require them (such as certain differential expression methods).

The preprocessing pipeline then follows the standard Scanpy workflow:

**Total-count normalization** scales all cells to a common library size, removing the confounding effect of variable transcript capture efficiency. **Log-plus-one transformation** is applied to compress the dynamic range and stabilize variance. **PCA** reduces the normalized expression matrix to its principal components, capturing the dominant axes of transcriptional variation. A **neighbor graph** is then built from the PCA coordinates, connecting cells that are transcriptionally similar. **UMAP** produces a 2D embedding of the neighbor graph for visualization. Finally, **Leiden clustering** partitions the neighbor graph into transcriptionally distinct communities representing candidate cell types or cell states.

## 4. UMAP and Spatial Visualization

### 4.1 UMAP Visualization

Three UMAP scatter plots are produced side by side, colored by total counts, number of unique genes detected, and Leiden cluster identity respectively. This panel serves to verify that the UMAP structure reflects transcriptional biology rather than technical gradients — clusters should not align strongly with sequencing depth metrics, which would indicate technical confounding. It also provides an overview of the transcriptional landscape and the number and separation of clusters identified.

### 4.2 Spatial Scatter Visualization

Leiden cluster labels are then projected back onto the physical tissue coordinates using Squidpy's `spatial_scatter()` function. Because Xenium data is at single-cell resolution rather than spot resolution, the `shape=None` parameter is specified — this instructs the plotting function to render each cell as a point at its centroid coordinate rather than as a predefined geometric shape (such as the hexagonal spots used in Visium). The resulting spatial plot reveals how the transcriptionally defined Leiden clusters are distributed across the lung cancer tissue section, showing whether different cell populations are spatially segregated or intermingled.

## 5. Spatial Statistics

### 5.1 Spatial Neighbor Graph Construction

All graph-based spatial analyses require a connectivity matrix encoding which cells are spatially adjacent. The graph is built using `squidpy.gr.spatial_neighbors()` with two important parameter choices specific to Xenium data:

- **`coord_type="generic"`:** Specifies that the spatial coordinates are arbitrary continuous 2D coordinates (as opposed to the regular grid structure of Visium spots), which is appropriate for single-cell resolved data where cell positions are determined by segmentation rather than a fixed array.

- **`delaunay=True`:** Constructs the neighbor graph using **Delaunay triangulation** — a computational geometry method that connects each cell to its geometrically natural nearest neighbors by finding the triangulation that maximizes the minimum angle of all triangles. This produces a graph that closely mirrors the actual physical adjacency structure of cells in tissue, without requiring a fixed radius or k-nearest-neighbor count.

---

### 5.2 Centrality Scores

**Concept:** Centrality scores are descriptive statistics of the spatial graph that characterize, for each cluster, how that cluster's cells are positioned and connected within the overall tissue graph. Three scores are computed:

- **Closeness centrality:** Measures how close, on average, a cluster's cells are to all other cells in the graph. Clusters with high closeness centrality are spatially central and well-connected across the tissue, while clusters with low closeness centrality tend to be peripheral or isolated.

- **Degree centrality:** Measures the fraction of cells from other clusters that are directly connected (i.e., adjacent neighbors) to cells in the focal cluster. A high degree centrality for a cluster means it has many neighbors from other clusters — indicating a boundary or interface position within the tissue.

- **Clustering coefficient:** Measures the degree to which neighboring cells of a given cluster also connect to one another. A high clustering coefficient indicates that a cluster's cells tend to form dense, compact groupings, while a low coefficient indicates they are more dispersed throughout the tissue.

**Execution and output:** Centrality scores are computed for all Leiden clusters using `squidpy.gr.centrality_scores()` and visualized together in a multi-panel bar plot using `squidpy.pl.centrality_scores()`. This reveals which clusters occupy central versus peripheral positions in the tissue, which clusters form tight aggregates versus diffuse distributions, and which clusters are most connected to other cell types — all of which carry biological meaning in the context of tumor tissue organization.

---

### 5.3 Co-occurrence Analysis

**Concept:** Co-occurrence analysis quantifies how the probability of observing one cluster changes as a function of distance from another cluster, computed directly from continuous spatial coordinates rather than the discrete neighbor graph. The score at any given radius is the ratio of the conditional probability of observing cluster *exp* near cluster *cond* to the unconditional probability of observing cluster *exp* at that scale. A score above 1 indicates spatial association; a score of 1 indicates spatial independence; a score below 1 indicates spatial exclusion.

**Subsampling:** Because the full dataset of 161,000 cells would require constructing a 161,000 × 161,000 distance matrix — computationally prohibitive — a 50% random subsample of cells is created using Scanpy's `pp.subsample()`. This subsampled AnnData object is stored as a new table in the SpatialData object (`sdata.tables["subsample"]`) and used for all co-occurrence computations. Squidpy automatically detects that the full pairwise distance computation would be memory-prohibitive and splits it into 39 sub-batches to manage memory usage.

**Execution and visualization:** Co-occurrence scores are computed across increasing radii for all cluster pairs and visualized as line plots for cluster 12 as the conditioning cluster. A companion spatial scatter plot of the subsampled cells colored by Leiden cluster is shown alongside, enabling direct visual comparison between the quantitative co-occurrence curves and the actual spatial distribution of clusters in the tissue.

---

### 5.4 Neighborhood Enrichment Analysis

**Concept:** Neighborhood enrichment operates on the discrete spatial connectivity graph to determine which pairs of Leiden clusters are more (or less) frequently adjacent than expected under a random permutation of cluster labels. The analysis generates a z-score for each cluster pair — positive z-scores indicate enriched co-adjacency (the two clusters tend to be neighbors), while negative z-scores indicate depletion (the clusters tend to avoid each other spatially).

**Execution:** The enrichment score is computed using `squidpy.gr.nhood_enrichment()` with 1,000 permutations, which randomize cluster labels across cells while preserving the graph structure to build the null distribution.

**Visualization:** Results are presented as a symmetric heatmap matrix of z-scores using `squidpy.pl.nhood_enrichment()`, displayed side by side with a spatial scatter plot of the subsampled tissue to aid interpretation. Cluster pairs with high positive scores represent populations that are physically adjacent in the tumor tissue — potentially reflecting cell-cell interactions, tumor-stroma boundaries, or immune infiltration patterns. Cluster pairs with strongly negative scores represent populations that are spatially segregated within the tissue section.

---

### 5.5 Spatially Variable Gene Detection with Moran's I

**Concept:** Moran's I is a spatial autocorrelation statistic that quantifies whether a gene's expression across cells shows non-random spatial structure. A Moran's I value close to +1 indicates strong positive spatial autocorrelation — cells with high expression of that gene tend to cluster together in space. A value near 0 indicates random spatial distribution. A negative value indicates spatial dispersion.

**Execution:** A new spatial neighbor graph is constructed on the subsampled dataset using Delaunay triangulation. Moran's I is then computed for all 480 genes across the subsampled cells using `squidpy.gr.spatial_autocorr()` with 100 permutations. Results are stored in `adata_subsample.uns["moranI"]` and sorted by Moran's I statistic in descending order.

**Top spatially variable genes:**

| Gene | Moran's I |
|---|---|
| AREG | 0.696 |
| MET | 0.683 |
| ANXA1 | 0.667 |
| EPCAM | 0.633 |
| DMBT1 | 0.588 |
| IGKC | 0.549 |
| IGHG1 | 0.518 |
| IDO1 | 0.496 |
| SPARC | 0.446 |
| APOE | 0.437 |

All top-ranked genes show strongly positive Moran's I values with adjusted p-values of zero, confirming highly significant and robust spatial patterning. The biological identities of these genes are highly informative in the context of lung cancer: **EPCAM** is a canonical epithelial cell marker, **AREG** (Amphiregulin) is an EGFR ligand associated with epithelial growth, **MET** is a receptor tyrosine kinase frequently amplified in lung cancer, **IGKC** and **IGHG1** are immunoglobulin subunit genes indicative of B cell and plasma cell presence, **IDO1** is an immune checkpoint-related enzyme associated with immunosuppression in the tumor microenvironment, and **APOE** is a lipid metabolism gene expressed by macrophages and other immune cells.

**Spatial visualization — two approaches:**

*Squidpy approach:* The top two spatially variable genes — AREG and MET — are mapped directly onto the cell coordinates using `squidpy.pl.spatial_scatter()` with `img=False` to show only the expression points without an underlying image. This confirms that both genes exhibit strongly localized spatial expression patterns consistent with their high Moran's I scores.

*SpatialData-plot approach:* The same genes are then visualized using the `spatialdata-plot` package, which renders gene expression colored circles directly overlaid on the **morphology focus image**. This approach provides the full anatomical context of the tissue morphology alongside the expression data, enabling direct visual correlation between the microscopy image and molecular measurements. Each gene is plotted in a separate figure panel at the full tissue scale.

## 6. Interactive Visualization with napari-spatialdata

The tutorial concludes by introducing **napari-spatialdata**, a plugin for the napari multi-dimensional image viewer that provides a fully interactive graphical interface for exploring SpatialData objects. By passing the SpatialData object to the `Interactive()` constructor, users can pan and zoom through the tissue image, toggle visibility of different data layers (images, cell boundaries, transcript points, expression overlays), and interactively explore gene expression patterns — such as AREG expression across all cells — without writing any additional code. This interactive component is particularly valuable for exploratory analysis of the large and complex spatial datasets that Xenium produces, where static plots at full tissue scale may not reveal fine-grained local patterns visible only at higher zoom levels.

## Summary and Conclusions

This analysis established a comprehensive end-to-end workflow for Xenium in situ transcriptomics data using the Squidpy and SpatialData ecosystem. The workflow addressed the unique challenges of Xenium data — principally its very large scale (161,000 cells, 40 million transcripts) and its subcellular resolution — through the use of on-disk Zarr storage, strategic subsampling for computationally intensive steps, and Delaunay triangulation-based spatial graphs appropriate for irregular single-cell coordinate systems.

The analytical pipeline moved through four major stages. Quality control confirmed extremely low background noise from control probes and guided conservative cell and gene filtering. Standard preprocessing and Leiden clustering identified transcriptionally distinct cell populations projected both in UMAP and physical tissue space. Spatial statistics — encompassing centrality scores, co-occurrence analysis, and neighborhood enrichment — characterized the spatial organization, adjacency structure, and scale-dependent associations among cell clusters across the tumor tissue. Finally, Moran's I-based spatially variable gene detection identified genes with the strongest spatial patterning, including markers of epithelial cells (EPCAM, AREG, MET), immune cells (IGKC, IGHG1, IDO1), and stromal cells (SPARC, APOE) — collectively reflecting the cellular heterogeneity of the lung cancer tumor microenvironment.

The dual visualization approach — combining Squidpy's coordinate-based scatter plots with SpatialData-plot's image-overlay rendering and napari's interactive GUI — demonstrates that the SpatialData ecosystem provides a flexible and scalable platform for multi-modal spatial transcriptomics analysis at single-cell resolution.
